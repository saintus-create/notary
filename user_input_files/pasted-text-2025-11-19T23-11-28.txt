Why this happens

Vite’s error message:

Could not load /usr/local/build/src/lib/components/Header.svelte (imported by src/routes/+page.svelte): ENOENT: no such file or directory

means that when Vite walked the import tree, it resolved the absolute path to
.../src/lib/components/Header.svelte

but the file was not there. The most common culprit on CI is a case mismatch:

# On your local machine (case‑insensitive)
src/lib/components/Header.svelte   ← exists
src/lib/components/header.svelte   ← maybe the actual file

# On CI (Linux, case‑sensitive)
src/lib/components/Header.svelte   ← does NOT exist

Because Windows/macOS allow both names to coexist, you never noticed.
The CI runner (Docker, GitHub Actions, etc.) sees only one of them, so it throws
ENOENT

.

Quick fix – the “case‑cure” recipe

Locate the actual file on your machine.

ls -l src/lib/components/

Rename it to match the import.

mv src/lib/components/header.svelte src/lib/components/Header.svelte

Update any imports that refer to the old name (if you changed the name).

<!-- src/routes/+page.svelte -->
<script>
  import Header from '$lib/components/Header.svelte';
</script>

Commit and push.

git add src/lib/components/Header.svelte
git commit -m "Rename Header component to correct case"
git push

Re‑run CI and it should pass.

Additional debugging tips

Add a console log in your
src/routes/+page.svelte

:

<script>
  import Header from '$lib/components/Header.svelte';
  console.log('Header loaded', Header);
</script>

If you run
npm run dev

locally, you’ll see the component in the console, proving the import resolves.

Run a full clean build locally to mimic CI:

rm -rf node_modules .svelte-kit
npm ci
npm run build

Check Vite’s resolved paths (in
vite.config.js

add a plugin):

import { defineConfig } from 'vite';
import { sveltekit } from '@sveltejs/kit/vite';
import fs from 'fs';

export default defineConfig({
  plugins: [
    sveltekit(),
    {
      name: 'log-headers',
      resolveId(source, importer) {
        if (source.endsWith('Header.svelte')) {
          console.log(`Resolving ${source} from ${importer}`);
        }
        return null;
      }
    }
  ]
});
